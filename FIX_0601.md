# VEGA 0.1.2 macOS 배포 디버깅 기록 (2026-06-01)

## 요약

2026-06-01 배포 테스트에서 다른 사용자 맥(Ted MacBook Air) 설치 시 다음 문제가 순차적으로 발생했다.

1. OpenRouter API 키 등록 시 `CERTIFICATE_VERIFY_FAILED`
2. 새 DMG 재설치 후에도 같은 SSL 오류 재현
3. 이후 `localhost:8100/api/health` 연결 실패
4. `launchctl`에서 `vega-backend`가 반복 재시작되며 `last exit code = 255`
5. 직접 실행 시 PyInstaller bootloader가 Python shared library 로딩 실패

최종 원인은 하나가 아니라 세 가지가 겹친 상태였다.

- PyInstaller 번들 환경에서 Python SSL이 시스템 CA를 못 찾아 외부 HTTPS 검증 실패
- 기존 `LaunchAgent`/백엔드 프로세스가 남아 새 앱을 설치해도 오래된 백엔드에 계속 연결
- Developer ID 수동 재서명 과정에서 entitlements가 빠져 hardened runtime의 library validation이 PyInstaller 내부 `Python.framework` 로딩을 차단

최종 해결 DMG:

- `build_output/VEGA-0.1.2.dmg`
- SHA-256: `fc9bb9aba4f6f79d1f6d08263e9fcc4709468acc848a15129bf7f2531ce815cd`
- Apple Notary: Accepted
- Staple: validate 완료
- Gatekeeper: Notarized Developer ID accepted

## 증상과 판단

### 1. OpenRouter API 키 등록 SSL 실패

설치 마법사에서 OpenRouter API 키를 입력하면 다음 오류가 표시됐다.

```text
연결 실패: <urlopen error [SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed:
unable to get local issuer certificate (_ssl.c:1081)>
```

개발자 머신의 `VEGA.app`에서는 재현되지 않았지만, 다른 사용자 맥의 깨끗한 설치 환경에서 발생했다.

판단:

- 개발자 머신은 기존 keychain, 시스템 CA, 로컬 Python 환경 영향을 받는다.
- 배포본은 PyInstaller onefile 백엔드가 자체 Python 런타임을 쓰므로 새 사용자 맥에서 CA 탐색 경로가 다를 수 있다.
- OpenRouter 키 등록 경로는 `web/routers/onboarding.py::_verify_key()`의 `urllib.request.urlopen()`이었다.

수정:

- `web/routers/onboarding.py`에 `certifi` 기반 `_ssl_context()` 추가
- OpenRouter/OpenAI/Anthropic 키 검증 `urlopen()`에 `context=_ssl_context()` 적용
- local OpenAI-compatible `/models` reachable check도 같은 SSL context 사용
- `bin/vega_backend_launcher.py`에서 프로세스 시작 시 `SSL_CERT_FILE`과 `REQUESTS_CA_BUNDLE`을 `certifi.where()`로 고정

관련 파일:

- `web/routers/onboarding.py`
- `bin/vega_backend_launcher.py`

### 2. OpenRouter 검증 URL 수정

처음에는 OpenRouter 키 검증에 `/api/v1/models`를 사용했다. 그런데 이 endpoint는 키 없이도 200을 반환할 수 있어 키 검증으로 부적절했다.

수정:

```text
https://openrouter.ai/api/v1/models
→ https://openrouter.ai/api/v1/key
```

검증:

```text
invalid-test-key → HTTP 401
```

관련 파일:

- `web/routers/onboarding.py`

### 3. 새 DMG 설치 후에도 이전 SSL 오류가 계속 재현

새 DMG를 설치했는데도 같은 SSL 오류가 계속 나왔다. 원인은 앱이 새 백엔드가 아니라 기존 백엔드에 붙고 있었기 때문이다.

기존 구조:

- `VEGA.app` 첫 실행 시 `~/Library/LaunchAgents/com.unohee.vega-backend.plist`를 복사/등록
- 이미 plist가 있으면 갱신하지 않음
- 이미 등록된 LaunchAgent가 있으면 `bootstrap` 오류를 무시
- UI는 `localhost:8100` 포트만 확인하고, 열려 있으면 백엔드 준비 완료로 판단

따라서 이전 백엔드가 8100을 계속 잡고 있으면 새 앱 설치 후에도 오래된 백엔드에 붙는다.

수정:

- `desktop/src/lib.rs::ensure_launchagent()`가 매 실행마다 Resources의 plist를 `~/Library/LaunchAgents`로 갱신
- 기존 LaunchAgent를 `bootout`
- 새 plist로 `bootstrap`
- `kickstart -k`로 현재 `/Applications/VEGA.app/Contents/MacOS/vega-backend` 실행 강제
- `wait_and_navigate()` 호출을 LaunchAgent 갱신 이후로 이동

관련 파일:

- `desktop/src/lib.rs`

### 4. `localhost:8100/api/health` 연결 실패

위 패치 이후 일부 사용자 환경에서 `localhost:8100/api/health` 연결 실패가 발생했다.

`launchctl` 상태:

```text
state = spawn scheduled
runs = 26
last exit code = 255
program = /Applications/VEGA.app/Contents/MacOS/vega-backend
```

처음에는 LaunchAgent 재등록 실패 가능성을 봤지만, 직접 실행 로그로 최종 원인이 확인됐다.

추가 보강:

- `wait_and_navigate()` 대기 시간을 30초에서 120초로 확장
- LaunchAgent bootstrap/kickstart 실패 시 앱이 `Contents/MacOS/vega-backend`를 직접 `spawn()`하는 fallback 추가
- 오류 페이지에 `/tmp/vega-backend.stderr.log` 확인 안내 추가

관련 파일:

- `desktop/src/lib.rs`

### 5. PyInstaller Python shared library 로딩 실패

Ted 맥에서 `/Applications/VEGA.app/Contents/MacOS/vega-backend` 직접 실행 시 다음 오류가 나왔다.

```text
[PYI-30816:ERROR] Failed to load Python shared library
'/var/folders/.../_MEI.../Python'
...
code signature ... Python.framework/Versions/3.14/Python not valid for use in process:
mapping process and mapped file (non-platform) have different Team IDs
```

이 에러는 macOS hardened runtime의 library validation이 PyInstaller가 임시 폴더에 푼 `Python.framework` 로딩을 막는 경우다.

원인:

- `desktop/entitlements.plist`에는 이미 `com.apple.security.cs.disable-library-validation`이 있었다.
- 하지만 `macstudio`에서 수동으로 `codesign`할 때 `--entitlements`를 넘기지 않아 최종 `vega-backend` 서명에서 entitlement가 사라졌다.
- 결과적으로 Developer ID 서명된 `vega-backend`가 다른 Team ID의 Python framework를 `dlopen()`하지 못했다.

수정:

- `macstudio` 수동 서명 시 `vega-backend`, `vega-desktop`, `VEGA.app` 모두에 `--entitlements /tmp/vega-notarize/entitlements.plist` 명시
- 서명 후 `codesign -d --entitlements :-`로 `disable-library-validation` 포함 여부 확인

검증된 entitlement:

```xml
<key>com.apple.security.cs.disable-library-validation</key>
<true/>
```

관련 파일:

- `desktop/entitlements.plist`
- `desktop/src/lib.rs`

## 최종 수정 목록

### `bin/vega_backend_launcher.py`

- PyInstaller 백엔드 시작 시 `certifi` CA 경로를 표준 SSL 환경변수에 설정

```python
import certifi
ca_bundle = certifi.where()
os.environ.setdefault("SSL_CERT_FILE", ca_bundle)
os.environ.setdefault("REQUESTS_CA_BUNDLE", ca_bundle)
```

### `web/routers/onboarding.py`

- certifi 기반 SSL context 추가
- API key 검증 `urlopen()`에 context 적용
- OpenRouter 검증 endpoint를 `/api/v1/key`로 변경

### `desktop/src/lib.rs`

- 기존 LaunchAgent를 매 실행마다 갱신/재등록
- `bootout` → `bootstrap` → `kickstart -k`
- UI가 8100에 붙기 전에 LaunchAgent 갱신 수행
- 백엔드 대기시간 120초로 확장
- LaunchAgent 실패 시 직접 `vega-backend` 실행 fallback 추가

### 코드 서명/공증 절차

- `vega-backend`와 `vega-desktop`, `VEGA.app` 서명 시 entitlements 명시
- DMG 서명 후 notarytool 제출
- staple 및 Gatekeeper 검증

## 사용한 주요 디버깅 명령

사용자 맥에서 백엔드 상태 확인:

```bash
launchctl print gui/$(id -u)/com.unohee.vega-backend
cat /tmp/vega-backend.stderr.log
cat /tmp/vega-backend.stdout.log
/Applications/VEGA.app/Contents/MacOS/vega-backend
echo EXIT:$?
```

서명/Gatekeeper 확인:

```bash
codesign --verify --deep --strict --verbose=4 /Applications/VEGA.app
spctl -a -vvv -t exec /Applications/VEGA.app
```

앱/백엔드 entitlements 확인:

```bash
codesign -d --entitlements :- /Applications/VEGA.app
codesign -d --entitlements :- /Applications/VEGA.app/Contents/MacOS/vega-backend
```

기존 LaunchAgent 정리:

```bash
launchctl bootout gui/$(id -u)/com.unohee.vega-backend 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.unohee.vega-backend.plist
pkill -f vega-backend 2>/dev/null || true
```

## 최종 빌드/공증 절차

로컬에서 앱/DMG 스테이지 생성:

```bash
scripts/build_dmg.sh
hdiutil create -volname 'VEGA 0.1.2' \
  -srcfolder build_output/dmg_stage \
  -ov -format UDZO build_output/VEGA-0.1.2.dmg
```

`macstudio`로 스테이지와 entitlements 전송:

```bash
rsync -a --delete build_output/dmg_stage/ macstudio:/tmp/vega-notarize/dmg_stage/
rsync -a desktop/entitlements.plist macstudio:/tmp/vega-notarize/entitlements.plist
```

`macstudio`에서 서명/공증:

```bash
WORK=/tmp/vega-notarize
KC=$WORK/vega-sign.keychain-db
NKC=$WORK/vega-notary.keychain-db
ENT=$WORK/entitlements.plist
APP=$WORK/dmg_stage/VEGA.app
DMG=$WORK/VEGA-0.1.2.dmg
ID=1D3C34D6CAEA18CA04265DCE3A4236C1F4992FCB

security list-keychains -d user -s "$KC" "$NKC" /System/Library/Keychains/SystemRootCertificates.keychain
security unlock-keychain -p vega-temp "$KC"
security set-key-partition-list -S apple-tool:,apple:,codesign: -s -k vega-temp "$KC" >/dev/null

codesign --force --options runtime --entitlements "$ENT" --keychain "$KC" --sign "$ID" "$APP/Contents/MacOS/vega-backend"
codesign --force --options runtime --entitlements "$ENT" --keychain "$KC" --sign "$ID" "$APP/Contents/MacOS/vega-desktop"
codesign --force --deep --options runtime --entitlements "$ENT" --keychain "$KC" --sign "$ID" "$APP"
codesign --verify --deep --strict --verbose=2 "$APP"

codesign -d --entitlements :- "$APP/Contents/MacOS/vega-backend" 2>/dev/null | grep -A1 "disable-library-validation"
codesign -d --entitlements :- "$APP" 2>/dev/null | grep -A1 "disable-library-validation"

rm -f "$DMG"
hdiutil create -volname "VEGA 0.1.2" -srcfolder "$WORK/dmg_stage" -ov -format UDZO "$DMG"
codesign --force --keychain "$KC" --sign "$ID" "$DMG"
codesign --verify --verbose=2 "$DMG"

security unlock-keychain -p vega-notary "$NKC"
xcrun notarytool submit "$DMG" --keychain "$NKC" --keychain-profile notarytool-intrect --wait
xcrun stapler staple "$DMG"
xcrun stapler validate "$DMG"
spctl -a -vvv -t install "$DMG"
shasum -a 256 "$DMG"
```

공증 완료 DMG를 로컬로 복사:

```bash
rsync -aE macstudio:/tmp/vega-notarize/VEGA-0.1.2.dmg build_output/
```

로컬 검증:

```bash
shasum -a 256 build_output/VEGA-0.1.2.dmg
hdiutil verify build_output/VEGA-0.1.2.dmg
```

## 배포 시 주의사항

- 기존 실패 빌드가 설치된 사용자에게는 새 DMG 설치 전 LaunchAgent 정리를 안내한다.
- PyInstaller onefile + hardened runtime 배포에서는 `disable-library-validation` entitlement가 반드시 필요하다.
- `codesign --force --deep`를 수동으로 다시 할 때 entitlements를 빼먹으면 기존 entitlements가 사라질 수 있다.
- `rsync`로 macOS 서명/스테이플된 DMG를 복사할 때는 `-aE`를 사용한다.
- app-specific password는 문서/로그/채팅에 남기지 않는다. 노출됐으면 Apple ID에서 폐기 후 재발급한다.
